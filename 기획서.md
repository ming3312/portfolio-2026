# 포트폴리오 기획서 (V3.2 - 라우팅 및 아카이브/상세 페이지 연동)

본 기획서는 사용자의 신규 피드백(라우팅 연동, 아카이브 페이지 구성, 황우 시그니처 상세 페이지의 프리미엄 감성 브랜딩 전략)을 반영하여 업데이트되었습니다.

---

## 1. 포트폴리오 개요

*   **목적**: 경력 이직 (웹 디자이너 & 퍼블리셔) + 프리랜서 외주 수주 및 비즈니스 홍보
*   **핵심 포지션**: 웹 디자이너 및 퍼블리셔 (디자인과 코딩을 아우르는 올라운더 웹 크리에이터)
*   **서브 역량**: 생성형 인공지능 API를 결합한 유체 풀스택 시스템 아키텍처 및 클라우드 배포 구축
*   **비주얼 컨셉**: **Hyper-Interactive Cyber Liquid (하이퍼 인터랙티브 사이버 리퀴드)**
*   **컬러 전략**: Matte Obsidian Black BG + Vivid Hot Pink 메인 (9%) + Neon Yellow 서브 (1%)

---

## 2. 페이지 라우팅 구조 & UX 흐름 (Routing & Journey)

포트폴리오의 구조는 SPA(Single Page Application) 환경 하에 가상 라우팅을 연동하여 매끄러운 화면 전환을 유지합니다.

```
[Main 가로 스크롤 카드] 
   ├── 개별 프로젝트 카드 클릭  ➡️  /works/:id (프로젝트 상세 페이지)
   └── [VIEW ALL WORKS] 클릭    ➡️  /works (아카이브 목록 페이지)
```

### 1) 상세 페이지 전환 인터랙션
*   메인 휠 가로 트랙에서 개별 작품 카드를 클릭할 때, 전역 마그네틱 커서가 즉시 흡입되며 화면 전체가 어두워진 후(Fade-Out), 선택한 카드의 썸네일 영역이 3D 폴딩 기법으로 열리며 `/works/:id`로 이동합니다.

### 2) 아카이브 페이지 전환 인터랙션
*   Works의 마지막 카드인 `VIEW ALL WORKS`를 호버하면 화살표 아이콘이 팽팽하게 튕기는 탄성 모션을 보여주며, 클릭 시 화면이 세로로 슬라이딩 다운되는 유기적 공간 스크롤과 함께 `/works` 아카이브로 이동합니다.

## 3. 아카이브 목록 페이지 기획 (`/works`)

아카이브 페이지는 대량의 프로젝트 히스토리를 효율적이고 직관적으로 탐색할 수 있도록 듀얼 뷰(Dual-View) 형태로 설계합니다.

```
+-------------------------------------------------------------+
|  [M i ng LOGO]                                [GNB Menu]   |
+-------------------------------------------------------------+
|                                                             |
|   A R C H I V E S                                           |
|                                                             |
|   [ VIEW TOGGLE:  ■ LIST VIEW  /  ▤ GRID VIEW ]            |
|                                                             |
|   +-----------------------------------------------------+   |
|   | 01. HWANGWOO BRANDING .......... Design & Publish   |   |
|   | 02. WORLD VISION CAMPAIGN A .... Directing & Build  |   |
|   | 03. WORLD VISION CAMPAIGN B .... Graphics & Publish |   |
|   +-----------------------------------------------------+   |
|                                                             |
+-------------------------------------------------------------+
```

### 1) 듀얼 뷰 토글 시스템 (Dual-View Switcher)
*   **LIST VIEW (기본형 - 최신순 정렬)**:
    *   *레이아웃*: 매거진 스타일의 정교한 1px 가로 디바이더 라인을 기본 골격으로 정보 정렬.
    *   *인터랙션*: 리스트 행 호버 시 해당 텍스트가 Vivid Hot Pink(#FF2D78)로 발광하며, 마우스 포인터를 따라 해당 프로젝트의 썸네일 이미지가 은은하고 가볍게 마우스를 추적하는 **"Hover Image Preview"** 효과를 적용하여 심미적 깊이감을 제공합니다.
*   **GRID VIEW (카드 썸네일형)**:
    *   *레이아웃*: 3열 또는 4열 바둑판식 카드 레이아웃으로, 모서리가 둥근 세련된 형태(Border-radius 24px)로 정렬.
    *   *인터랙션*: 마우스 호버 시 1px 실선 테두리가 흐릿한 네온 아우라로 Glowing 효과를 주며, 카드 내부 이미지가 미세하게 패럴랙스 줌인되는 모션 구현.

### 2) 기술 구현 연동
*   리스트 뷰의 각 텍스트 행과 그리드 뷰의 썸네일 카드는 모두 마그네틱 효과를 받으며, 클릭 시 일관되게 `/works/:id` 상세 케이스스터디 페이지로 리디렉션됩니다.

## 4. 프로젝트 상세 페이지 기획 (`/works/:id` - 황우 시그니처 쇼룸)

황우 쇼룸(https://hwangwoo001.cafe24.com/) 브랜딩/퍼블리싱 사례를 탑재하고 명품 한우 브랜딩 특유의 럭셔리하고 묵직한 감성을 표현하기 위해 최상의 UI 스토리라인을 적용합니다.

### 1) 비주얼 및 카피라이팅 기획
*   **헤드라인**: `THE ESSENCE OF LUXURY BEEF: HWANGWOO`
*   **디자인 컨셉**: **Classic Luxury Premium Showcase**
    *   어둡고 묵직한 카본 블랙(#050508) 배경 위에 샴페인 골드(Champagne Gold - Neon Yellow `#FFE000`를 활용)와 등육질의 핫핑크(#FF2D78)를 극소 배색하여 숭고하고 품격 있는 감성을 전달합니다.

### 2) 상세 스토리라인 3단계 레이아웃

#### **[Phase 1] The Origin (본질과 타협하지 않는 가치)**
*   **비주얼**: 안심(Tenderloin), 등심·채끝(Sirloin & Striploin) 등 황우의 시그니처 육질 마블링을 고해상도 그래픽 비주얼로 전면 패럴랙스 배치.
*   **내용**: "신뢰할 수 있는 안심, 깊은 풍미의 등심과 채끝. 황우가 지켜온 본질의 가치"를 서사적인 카피라이팅으로 풀어씀으로써, 단순 쇼핑몰 구축을 넘어 프리미엄 브랜딩 컨설턴트로서의 역량을 각인.

#### **[Phase 2] Interactive Digital Showroom (디지털 공간에 재현한 명품)**
*   **레이아웃**: 3D 깊이감(Z축)을 살린 비대칭 레이어에 황우 제품 목업과 브랜딩 패키지를 플로팅하여 노출.
*   **인터랙션**:
    *   황우 고유의 마블링 패키지 카드 호버 시 튕기듯 탄력 있게 반응하는 Framer Motion 기반 **물리 스프링 모션(Physics-based Tilt)** 적용.
    *   정교한 1px 라인 탭 구조를 탭핑하면 안심/등심 등 세부 부위별 고해상도 비주얼 컨텐츠가 부드럽게 크로스페이드되는 쇼룸 인터페이스 구현.

#### **[Phase 3] Technical Engineering Insight (기술적 가치 입증)**
*   **역할 및 성과**: 기획부터 디자인, 퍼블리싱까지 단독 빌드로 구축한 성과 요약.
*   **트러블슈팅 스토리**:
    *   사용자의 등급 및 부위 선택 시 실시간으로 고해상도 3D 비주얼 패키지 렌더링을 딜레이 없이 제공하기 위해, 리소스 로딩 파이프라인 최적화 및 브라우저 메모리 관리 문제를 어떻게 논리적으로 풀어나갔는지 기술적 인사이트 제시.
    *   배지 태그: `Branding`, `Interaction`, `Layout`, `Publishing`

## 5. 구현 검증 계획

*   **라우팅 지연성 테스트**: 페이지 이동(`/works` 및 `/works/:id`) 시 WebGL 리퀴드 캔버스의 소멸과 재생성이 버벅임 없이 60fps를 준수하는지 프레임 드랍 검증.
*   **반응형 레이아웃 확인**: 아크뷰 듀얼 뷰 토글 모션이 모바일 포트레이트 뷰에서도 그리드 압축 현상 없이 유연하게 적응하도록 세로 리스트 스케일링 강제 검사.
